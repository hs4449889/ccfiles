## 調査タスクの実行方針

### Gemini CLIを使った並列調査

調査や情報収集が必要なタスクでは、**Gemini CLI (`gemini`)** を活用して質の高い調査を行います。

#### 基本原則

1. **テーマごとに並列で切り出す**: 調査対象が複数のテーマに分かれる場合、それぞれを独立したGemini CLIプロセスとして並列実行する
2. **専門性の分離**: 各テーマは専門的な観点から深く調査する
3. **結果の統合**: 並列調査の結果を統合し、包括的な回答を構築する

#### 実行パターン

```bash
# 例: 3つのテーマを並列で調査
gemini -p "テーマA について詳しく調査してください" > /tmp/research_a.md &
gemini -p "テーマB について詳しく調査してください" > /tmp/research_b.md &
gemini -p "テーマC について詳しく調査してください" > /tmp/research_c.md &
wait

# 結果を統合して回答を構築
cat /tmp/research_*.md
```

#### 調査を並列化すべきケース

- **技術選定**: 複数のライブラリ/フレームワークを比較検討する場合
- **ベストプラクティス調査**: 異なる観点（セキュリティ、パフォーマンス、保守性）での調査
- **エラー調査**: 複数の原因仮説を同時に検証する場合
- **ドキュメント調査**: 複数の公式ドキュメントやリソースを参照する場合
- **コードレビュー観点**: 異なるレビュー観点を並列で分析する場合

#### 調査クオリティのガイドライン

1. **具体的なプロンプト**: Gemini CLIには具体的で明確な質問を投げる
2. **コンテキスト付与**: 必要に応じてコードやエラーメッセージをプロンプトに含める
3. **検証**: 調査結果は可能な限り実際のコードや公式ドキュメントで検証する
4. **出典明記**: 調査結果には情報源を明記する

## 通知設定

### tmuxセッションごとの通知

Claude Codeの各種イベントでtmuxセッションに応じた通知を出します。

```bash
# 通知スクリプトの呼び出し
./scripts/tmux-notify.sh <type>
```

#### 通知タイプ

| タイプ | 用途 | 音 |
|--------|------|-----|
| `question` | Claudeが質問するとき | ロボットっぽいビープ音 |
| `attention` | 承認が必要なとき | ねえねえ音 |
| `complete` | タスク完了時 | ピロリン♪ |

#### セッション別カラー

tmuxセッション名に基づいて自動的に12色からカラーが割り当てられます。
複数のClaude Codeウィンドウを視覚的に区別できます。

## カスタムエージェント

### Gemini Researcher

`/agent gemini-researcher` で呼び出し可能なカスタムサブエージェント。
複雑な調査タスクをGemini CLIに委譲して実行します。

```
/agent gemini-researcher "React 19の新機能について調査してください"
```
